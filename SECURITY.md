# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| `main` branch | Yes |

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report via:

- **GitHub Security Advisories**: [Report a vulnerability](https://github.com/zhengjiewu39-lab/MedWear-Health-Analytics/security/advisories/new)
- Contact the repository maintainer privately

Include description, reproduction steps, affected version, and suggested fix if available.

## Response timeline

- Acknowledgment within 72 hours
- Initial assessment within 7 days
- Fix or mitigation as soon as reasonably possible

## Scope

In scope: auth bypass, injection, sensitive health data exposure, exploitable dependency issues.

Out of scope: DoS against personal demo deployments, issues requiring local machine access.

## Safe harbor

Good-faith security research following this policy will not be pursued.
